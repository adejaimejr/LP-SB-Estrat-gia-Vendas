# SB VENDAS — Prompt de Refinamento para Replit Design Mode

A primeira versão da landing page ficou muito boa! Agora preciso de ajustes específicos para elevar o nível de qualidade e aproximar mais do template de referência (Custklen CRM Landing Page). Seguem as correções organizadas por seção:

---

## 🔧 CORREÇÃO 1: HERO — Adicionar elementos flutuantes para criar profundidade

**Problema:** O hero está bonito mas muito "flat" comparado ao template de referência Custklen, que usa cards/badges flutuantes sobre e ao redor da foto principal para criar profundidade e dinamismo visual.

**Solução:** Adicionar 2-3 elementos flutuantes ao redor da foto da Samara no hero:

1. **Badge flutuante superior direito** (sobre/perto da foto):
   - Card pequeno com fundo `#1A1A1A`, borda `rgba(201, 168, 76, 0.2)`, border-radius 12px
   - Conteúdo: Ícone de gráfico crescente (TrendingUp) em verde `#4ADE80` + texto "+50% faturamento"
   - Animação: Float suave (translateY -8px a 8px) com `animation: float 3s ease-in-out infinite`

2. **Badge flutuante inferior** (abaixo/ao lado da foto):
   - Card pequeno estilo WhatsApp: fundo `#1E3A2F`, border-radius 12px
   - Conteúdo: Ícone WhatsApp verde + texto "Lead convertido ✓"
   - Animação: Float com delay diferente: `animation: float 3s ease-in-out infinite 1.5s`

3. **Badge flutuante esquerdo** (entre o texto e a foto):
   - Circulo com avatar pequeno + estrelas ⭐⭐⭐⭐⭐ + "4.9/5"
   - Fundo `#1A1A1A`
   - Animação: Float com outro delay

**CSS para animação float:**
```css
@keyframes float {
  0%, 100% { transform: translateY(0px); }
  50% { transform: translateY(-10px); }
}
```

Esses badges devem ter `position: absolute` em relação ao container da imagem, com `z-index` adequado. No mobile, esconder ou reposicionar.

## 🔧 CORREÇÃO 5: SEÇÃO RESULTADOS — Adicionar animação count-up nos números

**Problema:** Os números (+50%, 40%, +50, 24/7) estão estáticos. Uma animação de contagem ao entrar na viewport aumenta drasticamente o impacto.

**Solução:**
- Implementar IntersectionObserver que dispara count-up animation quando a seção entra na viewport
- Cada número conta de 0 até o valor final em ~2 segundos com easing
- Exemplo de implementação:
```javascript
// Count-up animation
const animateCounter = (element, target, suffix = '', prefix = '') => {
  let current = 0;
  const increment = target / 60; // ~60 frames in 2s at 30fps
  const timer = setInterval(() => {
    current += increment;
    if (current >= target) {
      current = target;
      clearInterval(timer);
    }
    element.textContent = prefix + Math.floor(current) + suffix;
  }, 33);
};

// Use IntersectionObserver to trigger when visible
```
- Para "24/7", não animar — simplesmente fazer fade-in
- Os separadores dourados entre os stats (linhas verticais e horizontais) devem ser um pouco mais visíveis: `border-color: rgba(201, 168, 76, 0.25)`

## 🔧 CORREÇÃO 10: NAVEGAÇÃO — Adicionar efeito glassmorphism no scroll

**Problema:** A navegação fixa parece não ter o efeito de transição entre transparente (no hero) e sólido escuro com blur (ao scrollar).

**Solução:**
- Implementar transição de navbar:
```javascript
// On scroll, add class to nav
window.addEventListener('scroll', () => {
  const nav = document.querySelector('nav');
  if (window.scrollY > 80) {
    nav.classList.add('nav-scrolled');
  } else {
    nav.classList.remove('nav-scrolled');
  }
});
```
```css
nav {
  background: transparent;
  transition: all 0.3s ease;
}
nav.nav-scrolled {
  background: rgba(10, 10, 10, 0.85);
  backdrop-filter: blur(20px);
  -webkit-backdrop-filter: blur(20px);
  border-bottom: 1px solid rgba(201, 168, 76, 0.1);
  box-shadow: 0 4px 30px rgba(0, 0, 0, 0.3);
}
```

## 🔧 CORREÇÃO 11: ANIMAÇÕES DE SCROLL — Adicionar reveal animations em todas as seções

**Problema:** As seções parecem aparecer estaticamente sem animações de entrada.

**Solução:**
- Implementar IntersectionObserver global para animar elementos ao entrarem na viewport:
```css
.reveal {
  opacity: 0;
  transform: translateY(30px);
  transition: opacity 0.6s ease, transform 0.6s ease;
}
.reveal.visible {
  opacity: 1;
  transform: translateY(0);
}
/* Stagger children */
.reveal-stagger > * {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 0.5s ease, transform 0.5s ease;
}
.reveal-stagger.visible > *:nth-child(1) { transition-delay: 0s; opacity: 1; transform: translateY(0); }
.reveal-stagger.visible > *:nth-child(2) { transition-delay: 0.1s; opacity: 1; transform: translateY(0); }
.reveal-stagger.visible > *:nth-child(3) { transition-delay: 0.2s; opacity: 1; transform: translateY(0); }
.reveal-stagger.visible > *:nth-child(4) { transition-delay: 0.3s; opacity: 1; transform: translateY(0); }
.reveal-stagger.visible > *:nth-child(5) { transition-delay: 0.4s; opacity: 1; transform: translateY(0); }
.reveal-stagger.visible > *:nth-child(6) { transition-delay: 0.5s; opacity: 1; transform: translateY(0); }
```
```javascript
const observer = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      entry.target.classList.add('visible');
    }
  });
}, { threshold: 0.15 });

document.querySelectorAll('.reveal, .reveal-stagger').forEach(el => observer.observe(el));
```
- Aplicar `.reveal` em: section labels, títulos, subtítulos, fotos
- Aplicar `.reveal-stagger` em: grids de cards (pain points, serviços), diferenciais, stats
